Detailed Description

Code to test out the basic activity monitor for a scenario with top-left/lower-right semantics of a tracked square in an image.

The image is "grayscale" and gets thresholded to generate a single region. That region is processed to generate a track point, which is used to establish activity state semantics. The implementation builds out a Monitor.

The code below

./activity01regions

runs the script.

Outcome

The square starts off in activity region 1, leaves the region into undefined space, enters activity region 2, then leaves it and stops at the path terminal point. The state output is purely text-based. There is visual output of the square to see where it is, and ultimately that it stops moving at the end of the path.
